# 北京美食分析与推荐系统 - 答辩文档

## 一、项目概述

### 1.1 项目背景
随着互联网餐饮行业的快速发展，用户对美食推荐和数据分析的需求日益增长。本系统基于Django框架开发，旨在为北京地区的美食爱好者提供全面的店铺信息查询、数据分析、智能推荐和可视化展示功能。

### 1.2 项目目标
- 构建一个功能完善的美食信息管理系统
- 提供多维度数据分析和可视化展示
- 实现基于用户行为的智能推荐算法
- 集成地图服务，提供地理位置可视化
- 支持评论情感分析，挖掘用户真实反馈

### 1.3 技术栈
- **后端框架**: Django 5.2.10
- **数据库**: MySQL 8.0+
- **前端技术**: HTML5, CSS3, JavaScript, ECharts 5.4.3
- **地图服务**: 百度地图API v3.0
- **Python版本**: Python 3.10+
- **其他**: SimpleUI (Django Admin美化), PyMySQL, jieba (中文分词), Redis (任务队列)

### 3.3 数据分析模块

#### 3.3.1 Top10排行榜算法

**综合人气榜算法** (`views.py:278-302`):
```python
def top10_popular(request):
    # 1. 获取最大值用于归一化
    max_comments = FoodShop.objects.aggregate(
        models.Max('comment_count')
    )['comment_count__max'] or 1
    max_score = FoodShop.objects.aggregate(
        models.Max('shop_score')
    )['shop_score__max'] or 1
    
    # 2. 归一化处理并计算综合人气分
    popular_shops = FoodShop.objects.all().annotate(
        normalized_comments=ExpressionWrapper(
            F('comment_count') / float(max_comments),
            output_field=FloatField()
        ),
        normalized_score=ExpressionWrapper(
            F('shop_score') / float(max_score),
            output_field=FloatField()
        ),
        popularity_score=ExpressionWrapper(
            F('normalized_comments') * 0.4 + F('normalized_score') * 0.6,
            output_field=FloatField()
        )
    ).order_by('-popularity_score')[:10]
```

**算法说明**:
1. **归一化处理**: 将评论数和评分归一化到[0,1]区间
   - `normalized_comments = comment_count / max_comments`
   - `normalized_score = shop_score / max_score`

2. **加权综合评分**:
   - `popularity_score = normalized_comments × 0.4 + normalized_score × 0.6`
   - 评分权重更高(0.6)，评论数权重较低(0.4)

3. **排序**: 按综合人气分降序排列，取前10名

**其他榜单**:
- **评分最高榜**: 按`shop_score`降序，`comment_count`作为次要排序
- **评论最多榜**: 按`comment_count`降序
- **人均最高榜**: 按`avg_price`降序
- **人均最低榜**: 按`avg_price`升序（性价比之选）
- **米其林推荐榜**: 筛选有米其林认证的店铺

### 3.4 推荐系统模块

#### 3.4.1 推荐算法设计

本系统实现了**混合推荐算法**，结合了协同过滤、内容相似度和热门推荐。

#### 3.4.2 算法1: ItemCF协同过滤

**算法原理**:
ItemCF (Item-based Collaborative Filtering) 基于物品的协同过滤算法。

**实现步骤** (`views.py:936-963`):
```python
# 1. 找到收藏了用户已收藏店铺的其他用户
similar_users_collects = ShopCollect.objects.filter(
    shop__shop_id__in=collected_shop_ids
).exclude(user=request.user).values('user_id').annotate(
    common_count=Count('id')
).filter(common_count__gte=2).order_by('-common_count')[:10]

# 2. 获取这些相似用户的ID
similar_users_ids = [item['user_id'] for item in similar_users_collects]

# 3. 统计这些用户还收藏了什么店铺（排除用户已收藏的）
if similar_users_ids:
    cf_recommendations = ShopCollect.objects.filter(
        user_id__in=similar_users_ids
    ).exclude(
        shop__shop_id__in=collected_shop_ids
    ).values('shop_id').annotate(
        recommend_count=Count('id')
    ).order_by('-recommend_count')[:10]
    
    # 4. 按共同收藏用户数排序推荐
    for rec in cf_recommendations:
        shop = FoodShop.objects.get(shop_id=rec['shop_id'])
        recommended_shops.append(shop)
        recommendation_reasons[shop.shop_id] = \
            f"有 {rec['recommend_count']} 位用户同时收藏了您喜欢的店铺"
```

**算法优势**:
- 能够发现用户可能喜欢但未注意到的店铺
- 基于用户行为数据，推荐结果更符合用户偏好
- 可解释性强，提供推荐理由

**适用场景**:
- 用户已有2+个收藏记录
- 存在其他用户有相似收藏行为

#### 3.4.3 算法2: 内容相似度推荐

**算法原理**:
基于店铺的属性特征（菜系、区县、价格）计算相似度。

**实现步骤** (`views.py:965-986`):
```python
# 1. 提取用户收藏店铺的特征
collected_types = set([s.food_type for s in collected_shops if s.food_type])
collected_counties = set([s.county for s in collected_shops if s.county])

# 2. 计算平均价格
valid_prices = [s.avg_price for s in collected_shops if s.avg_price]
if valid_prices:
    avg_collected_price = sum(valid_prices) / len(valid_prices)

# 3. 推荐同菜系同区县的店铺
if collected_types and collected_counties:
    content_similar = FoodShop.objects.filter(
        food_type__in=collected_types,
        county__in=collected_counties
    ).exclude(shop_id__in=collected_shop_ids).order_by('-shop_score')[:10]
    
    for shop in content_similar:
        if shop not in recommended_shops:
            recommended_shops.append(shop)
            recommendation_reasons[shop.shop_id] = \
                f"与您收藏的 {shop.food_type} 店铺相似（{shop.county}）"
```

**相似度计算维度**:
1. **菜系类型**: 完全匹配
2. **区县位置**: 完全匹配
3. **价格区间**: 在用户平均价格的±30%范围内

**算法优势**:
- 推荐结果更符合用户已知偏好
- 计算速度快，无需复杂的相似度计算
- 推荐理由清晰易懂

#### 3.4.4 算法3: 价格区间匹配

**算法原理**:
根据用户收藏店铺的平均价格，推荐价格相近的店铺。

**实现步骤** (`views.py:988-1000`):
```python

    # 推荐价格在 [avg_price*0.7, avg_price*1.3] 范围内的店铺
    price_similar = FoodShop.objects.filter(
        avg_price__gte=avg_collected_price * 0.7,
        avg_price__lte=avg_collected_price * 1.3
    ).exclude(shop_id__in=collected_shop_ids).order_by('-shop_score')[:10]
    
    for shop in price_similar:
```

**价格区间计算**:
- 下限: `avg_price × 0.7` (比平均价格低30%)
- 上限: `avg_price × 1.3` (比平均价格高30%)

**算法优势**:
- 符合用户消费能力
- 避免推荐超出预算的店铺

#### 3.4.5 算法4: 热门推荐（冷启动）

**算法原理**:
当用户没有收藏或推荐数量不足时，使用热门店铺作为补充。

**实现步骤** (`views.py:1002-1025`):

**算法优势**:
- 解决冷启动问题（新用户无收藏数据）
- 保证始终有推荐结果
- 推荐质量有保障（高评分+高评论数）

**作用**: 增强或减弱情感词的强度
- 例如: "非常好吃" = "好吃"(1.0) × "非常"(1.8) = 1.8

**3. 否定词处理** (`sentiment_core.py:38-46`):

```python
self.negators = {
    '不', '没', '无', '非', '未',
    '没有', '不会', '不能', '不可',
    '不太', '不太', '绝不', '绝非',
    # ... 更多否定词
}
```

**处理逻辑**:
- 检测到否定词后，将后续情感词的情感反转
- 否定后权重降低: `word_score = -word_score × 0.6`
- 例如: "不好吃" = -"好吃"(1.0) × 0.6 = -0.6

**4. 双重否定处理** (`sentiment_core.py:49-52`):

```python
self.double_negatives = {
    '不得不', '不能不', '不可不', '不会不',
    '不是不', '无不', '非不',
}
```

**处理逻辑**:
- 双重否定 = 肯定
- 例如: "不得不推荐" = 正面情感

**5. 转折词处理** (`sentiment_core.py:55-58`):

```python
self.conjunctions = {
    '但是', '可是', '然而', '不过', '却',
    '虽然', '尽管', '即使', '即便',
}
```

**处理逻辑**:
- 检测到转折词后，重置否定栈和权重
- 转折词后的内容权重更高

**6. 情感分析主算法** (`sentiment_core.py:194-292`):

```python
def analyze(self, text):
    """分析文本情感"""
    # 1. 预处理
    text = text.strip()
    words = jieba.lcut(text)  # 中文分词
    
    # 2. 初始化
    sentiment_score = 0.0
    effective_words = 0
    negation_stack = []
    
    # 3. 遍历分词结果
    i = 0
    while i < len(words):
        current_word = words[i]
        weight = 1.0
        
        # 检查双重否定
        if i + 1 < len(words):
            two_word = current_word + words[i + 1]
            if two_word in self.double_negatives:
                if len(negation_stack) > 0:
                    negation_stack.pop()  # 抵消一个否定
                i += 2
                continue
        
        # 检查否定词
        if current_word in self.negators:
            negation_stack.append(True)
            i += 1
            continue
        
        # 检查程度副词
        if current_word in self.intensifier_weights:
            weight = self.intensifier_weights[current_word]
            i += 1
            if i >= len(words):
                break
            current_word = words[i]
        
        # 检查转折词
        if current_word in self.conjunctions:
            negation_stack = []
            weight = 1.0
            i += 1
            continue
        
        # 计算情感得分
        if current_word in self.sentiment_dict:
            word_score = self.sentiment_dict[current_word] * weight
            
            # 应用否定词
            if negation_stack:
                word_score = -word_score * 0.6
                negation_stack.pop()
            
            sentiment_score += word_score
            effective_words += 1
        
        i += 1
    
    # 4. 归一化处理
    if effective_words > 0:
        normalized_score = sentiment_score / (effective_words * 1.2)
        final_score = 0.5 + normalized_score
        final_score = max(0.0, min(1.0, final_score))
    else:
        final_score = 0.5
    
    # 5. 情感分类
    if final_score > 0.65:
        sentiment = '正面'
    elif final_score < 0.35:
        sentiment = '负面'
    else:
        sentiment = '中性'
    
    return sentiment, round(final_score, 4)
```

**算法流程**:
1. **文本预处理**: 去除空白，使用jieba分词
2. **词性分析**: 识别情感词、程度副词、否定词、转折词
3. **情感计算**: 
   - 基础情感分 = 情感词权重 × 程度副词权重
   - 应用否定词: 反转并降低权重
   - 累加所有有效词的情感分
4. **归一化**: 将情感分归一化到[0,1]区间
5. **分类**: 
   - >0.65: 正面
   - <0.35: 负面
   - 其他: 中性

**算法优势**:
- 无需训练数据，基于规则和词典
- 计算速度快，适合批量处理
- 可解释性强，能追踪每个词的情感贡献
- 支持中文语境（否定词、程度副词、双重否定）

#### 3.5.3 批量情感分析任务

**任务队列系统** (`redis_queue.py`):

**架构设计**:
```
管理员创建任务 → Redis队列 → Worker进程处理 → 更新数据库
```

**关键组件**:

1. **RedisQueue类** (`redis_queue.py:12-106`):
   - 管理任务队列
   - 存储任务进度
   - 保存任务结果

2. **SentimentWorker类** (`redis_queue.py:108-236`):
   - 从Redis队列获取任务
   - 批量处理评论（每批500条）
   - 实时更新进度
   - 异常处理和错误记录

**处理流程**:
```python
def process_task(self, task_data):
    task_id = task_data['task_id']
    task_type = task_data.get('task_type', 'full')
    batch_size = task_data.get('batch_size', 500)
    
    # 1. 更新任务状态为"处理中"
    task = SentimentAnalysisTask.objects.get(task_id=task_id)
    task.mark_started()
    
    # 2. 根据任务类型获取评论
    if task_type == 'full':
        comments = FoodComment.objects.all()
    elif task_type == 'incremental':
        comments = FoodComment.objects.filter(sentiment__isnull=True)
    elif task_type == 'reanalyze':
        comments = FoodComment.objects.all()
    
    total = comments.count()
    processed = 0
    success = 0
    failed = 0
    
    # 3. 分批次处理
    for i in range(0, total, batch_size):
        batch = comments[i:i + batch_size]
        
        for comment in batch:
            try:
                # 分析情感
                sentiment, score = self.analyzer.analyze(comment.content)
                
                # 更新评论
                comment.sentiment = sentiment
                comment.sentiment_score = score
                comment.save()
                
                success += 1
            except Exception as e:
                failed += 1
        
        processed += len(batch)
        
        # 4. 更新进度（Redis + 数据库）
        self.queue.update_progress(task_id, processed, success, failed)
        task.update_progress(processed, total)
    
    # 5. 任务完成
    result_summary = {
        'total': total,
        'success': success,
        'failed': failed,
        'success_rate': success / total * 100 if total > 0 else 0
    }
    task.mark_completed(result_summary)
```

**任务类型**:
- **full**: 分析所有评论
- **incremental**: 只分析未分析的评论
- **reanalyze**: 重新分析已有评论
- **test**: 小批量测试

**性能优化**:
- 批量处理: 每批500条，减少数据库操作
- 异步处理: 使用Redis队列，不阻塞Web请求
- 进度实时更新: 通过Redis实时获取进度

### 7.4 项目难点与解决方案

#### 难点1: 推荐算法冷启动问题

**问题**: 新用户没有收藏数据，无法进行个性化推荐。

**解决方案**:
- 实现热门推荐算法作为补充
- 使用评分×评论数计算人气值
- 保证始终有推荐结果

#### 难点2: 情感分析准确性

**问题**: 中文语境复杂，否定词、程度副词影响情感判断。

**解决方案**:
- 构建全面的情感词典（正面200+，负面150+）
- 实现否定词、程度副词、双重否定、转折词处理
- 使用权重机制增强或减弱情感强度

#### 难点3: 大数据量性能优化

**问题**: 店铺和评论数据量大，查询和渲染性能差。

**解决方案**:
- 使用Django ORM的select_related和annotate优化查询
- 实现分页功能，只查询当前页数据
- 前端图表懒加载和数据采样

#### 难点4: 地图API加载失败

**问题**: 开启VPN时百度地图API无法加载。

**解决方案**:
- 实现15秒超时检测
- 提供友好的错误提示
- 说明需要关闭VPN
